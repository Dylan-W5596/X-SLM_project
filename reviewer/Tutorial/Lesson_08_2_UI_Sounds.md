# 第 8.2 課：UI 提示音訊 - 視覺之外的互動深度

這節課解析 **`frontend/src/utils/soundUtils.js`** 與它的應用。

---

## 1. Web Audio API：輕量的音效方案

在 React 中，我們不使用沉重的 `Video` 套件，而是直接使用原生瀏覽器的 `Audio` 物件：
```javascript
const audio = new Audio(path);
audio.play();
```
這非常省資源。

---

## 2. 音量與用戶體驗 (Volume Control)

```javascript
audio.volume = 0.3; // 設定為 30% 左右
```
**極深解析**：在專業 App 中，我們永遠不會把 volume 預設為 1.0 (100%)。
1.  **突兀感**：100% 的音量會蓋過使用者的音樂。
2.  **失真**：有些便宜喇叭在 100% 下會有爆音。
設定在 0.2 到 0.4 之間，能產生一種「若有似無」、像是真的在操控一台精密機器的清脆回饋感。

---

## 3. 按鈕點擊的防抖音效

在 `App.jsx` 中，幾乎每個按鈕處理函式的開頭都有 `playSound('click', ...)`。
這彌補了電子螢幕沒有「實體回彈感」的缺陷。

---

## 4. 控制開關的傳遞

```javascript
export const playSound = (type, enabled = true) => {
    if (!enabled) return;
    // ...播放
};
```
我們在工具函數裡就預留了 `enabled` 判斷。這代表如果使用者在設定裡關閉音效，我們不需要去改 50 個按鈕的代碼，音效自然就不會發出來。這就是程式設計中的 **「單一出口原則」**。

---

## 🔍 教學思考：
為什麼不把所有音效檔案都先載入到記憶體？
> **深度解答**：那叫 **「預載 (Preloading)」**。對於遊戲來說很重要，但對於聊天軟體來說，音效檔案很小。我們選擇在播放時才建立 `new Audio`，這樣可以節省 App 剛載入時的記憶體。除非你的使用者正在極慢的機械硬碟上執行，否則這幾毫秒的載入延遲是感覺不出來的。
